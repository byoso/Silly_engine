# Project context

Language: Python
Tests: pytest

# Nature of the project

silly_engine is a set of tools that can be used either as a global package or each module independently as a stand-alone. Therefore, it is important not to create dependencies between modules, even if it means duplicating code.

# Constraints:
- All comments in the code must be written in English, the code is always in English.
- Never use nested classes in Python.
- Never modify the tools contained in silly_engine without an explicit request.
- Unit tests only.
- keep the documentation up to date when the code changes.
